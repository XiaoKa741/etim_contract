# ETIM 项目需求规格说明书（已确认版）

> 本文档基于《ETIM项目设计文档.pdf》整理，补充了口头确认的细节，作为合约开发的最终参考。

---

## 一、代币总量与分配

**总量：2,100,000,000 ETIM，不增发。**

| 模块 | 数量 | 占比 | 说明 |
|------|------|------|------|
| 增长池 Growth Pool | 1,925,700,000 | 91.7% | 挖矿/参与/贡献释放 |
| 协议市场基础设施 Market Infra | 105,000,000 | 5% | POL / 做市仓 / 对冲仓 |
| 生态建设基金 | 21,000,000 | 1% | 支持应用生态发展 |
| 社区建设 | 21,000,000 | 1% | 地面社区工作室补贴、会议 |
| 空投 | 21,000,000 | 1% | 空投活动 |
| 以太坊基金会 | 6,300,000 | 0.3% | 合作 |

> Market Infra、生态建设基金、社区建设、空投、以太坊基金会共 8.3%，在部署时立即释放给对应地址。

**底池交易对**：ETIM / ETH（以太坊主网，Uniswap V4）。初始流动性部分锁死，不能 rug pull。

**关键设计原则**：用户获得的 ETIM 不从底池直接购买，而是通过智能合约从增长池中挖矿释放。

## 二、用户参与（入场规则）

### 2.1 入场前提

- 用户必须已被他人邀请（即 `referrerOf[用户地址] != address(0)`）。
- 每个地址只能入场一次（参与后不可重复）。

### 2.2 入场金额

- 范围：100U ～ 150U 等值 ETH（owner 可调整上下限）。
- 方式：直接向合约转入 ETH（`receive()` 或调用 `deposit()`）。

### 2.3 每日存款限额

- 每日可接受的新增存款上限 = LP 底池 ETH 总储备 × 20%（比例 owner 可调节）。
- **特殊规则**：当 `dailyDepositCap == 0` 时，以实时 LP ETH 储备 × 20% 作为当日限额（而非阻断所有存款）。

### 2.4 入场资金分配（合计 100%）

用户入场的 ETH 按以下比例分配，**全部由合约自动执行**：

| 份额 | 比例 | 操作 | 说明 |
|------|------|------|------|
| 节点业绩份 | 1% | ETH → swap → ETIM，分发给所有已激活节点 | 已激活节点均摊，作为业绩奖励 |
| LP 份 | 69% | ETH 的一半 swap 成 ETIM，连同另一半 ETH 一起注入底池 | 增加 ETIM/ETH 流动性 |
| 销毁份 | 25% | ETH → swap → ETIM → 发送至销毁地址 | 通缩机制 |
| 奖励份 | 5% | **保留为 ETH**，累积在合约中，由 owner 手动触发分发 | 见下方 2.5 节 |

#### 注入控制开关（储蓄池模式）

- **开关关闭（默认）**：LP 份（69%）**直接**注入底池。
- **开关开启**：LP 份先累积进储蓄池，由 owner 按设定的间隔时间和数量控制注入速率（抗波动工具）。

### 2.5 奖励份（5% ETH）分发规则

5% ETH 在合约中累积，**需要手动触发提取**（操作人给 gas）：

| 接收方 | 比例 | 说明 |
|--------|------|------|
| 已激活节点中等级 ≥ S2 的用户 | 50% | 均摊给所有达到 S2 及以上等级的激活节点持有者 |
| 基金会地址 | 30% | 直接发送到基金会 ETH 地址 |
| 奖池（prize pool） | 10% | 累积在合约指定地址，另行分配 |
| 官方钱包 | 10% | 直接发送到官方 ETH 地址 |

## 三、挖矿（增长池释放）

### 3.1 基础规则

- 用户入场后，每天可从增长池中挖出等值于入场金额 **0.1% ～ 1%**（可由 owner 调节）的 ETIM。
- 挖矿以**金本位（U 本位）**计算：每天产出的是"一定 USD 价值"对应的 ETIM 数量（按当日价格换算）。
- **总上限**：用户累计可挖出的 USD 价值 = 入场金额 USD 等值（+ 节点额度加成，见第四节）。
- 结算时区：UTC+0 每日结算。
- **懒结算机制**：ETIM 只在用户主动调用 `claim()` 时才实际链上转账；之前为链下记账。

### 3.2 个人等级加速

高等级用户每天额外多挖（加速不增加总上限，只是更快到达上限）：

| 等级 | 加速比例 | 实际日产出 = 基础日产出 × (1 + 加速%) |
|------|----------|---------------------------------------|
| S0 | +3% | 基础 × 1.03 |
| S1 | +7% | 基础 × 1.07 |
| S2 | +10% | 基础 × 1.10 |
| S3 | +12% | 基础 × 1.12 |
| S4 | +15% | 基础 × 1.15 |
| S5 | +18% | 基础 × 1.18 |
| S6 | +20% | 基础 × 1.20 |

## 四、会员等级

### 4.1 升级条件

同时满足以下三个条件才能升级（owner 可调整团队币量阈值）：

| 等级 | 直推人数（≥） | 个人持仓 ETIM（≥） | 团队币量 ETIM（≥，不含自己） |
|------|--------------|-------------------|------------------------------|
| S0 | 0 | 0 | 0 |
| S1 | 5 | 50,000 | 500,000 |
| S2 | 10 | 100,000 | 3,000,000 |
| S3 | 15 | 150,000 | 5,000,000 |
| S4 | 20 | 200,000 | 7,000,000 |
| S5 | 25 | 300,000 | 9,000,000 |
| S6 | 30 | 400,000 | 11,000,000 |

> **注**：文档原版 S3 团队阈值为 500 万，S4 为 700 万，S5 为 900 万，S6 为 1100 万——此处以文档原版为准。

### 4.2 等级计算规则

- 等级实时根据链上数据判断，不需要 owner 干预（用户持仓变化时自动触发重算）。
- 用户也可手动调用 `updateReferralLevel()` 刷新等级。
- 团队币量：**只统计到有嵌套（某成员自己达到同等或更高等级）时中止**，不无限向下穿透。

  > 示例：A→B→C，若 B 已是 S1，则 A 计算团队时不再把 C 的持仓算进去（B 的下线不穿透给 A）。

## 五、邀请绑定关系

### 5.1 绑定规则

**双向转账即建立父子关系**：
1. A 向 B 发送任意数量 ETIM → 记录"A 发给 B"
2. B 向 A 发送任意数量 ETIM → 检测到双向记录
3. **先发送的一方成为父（referrer）**，另一方成为子（invitee）

绑定一旦建立，不可更改。

### 5.2 团队关系

- `referrerOf[user]`：user 的直接上级。
- `referralsOfList[referrer]`：referrer 直接邀请的所有人。
- 团队计算：从用户往下递归，遇到与用户**同等级或更高等级**的成员时停止（该成员的下线不再计入）。

## 六、节点 NFT

### 6.1 基本信息

- **总量**：500 个，已在以太坊主网部署。
- **持有查询**：合约通过 ERC721 `balanceOf` 查询用户地址持有的节点数量，无需链上铸造/购买逻辑。

### 6.2 节点额度加成

- 每持有 1 个节点，挖矿总额度额外增加 **300U**。
- 即：总可挖 USD 价值 = 入场 USD + 300 × 节点数量。

### 6.3 节点激活与业绩奖励

- 节点持有者需达到 **≥ S1 等级**才能激活节点。
- 激活后，从全网每笔入场的 **1% ETH（swap 成 ETIM）**中均摊获得业绩分红。
- 节点持有者手动调用合约领取奖励。
- 节点数量变化时需调用 `syncNodes()` 同步状态，或在 `claimNodeRewards()` 时自动同步。

## 七、买卖规则

### 7.1 购买（Buy）

- **触发条件**：仅当增长池（Growth Pool）全部挖完后，才开放购买接口。
- **费率**：3% 买入税，全部归官方钱包（ETH 计）。

### 7.2 卖出（Sell）

- **随时可卖**，正常 swap。
- **卖出税率**：3%，按 ETIM 数量从用户卖出量中扣除，分配如下：

| 接收方 | 比例 | 说明 |
|--------|------|------|
| S6+ 用户奖励池 | 0.5% | 累积在合约，owner 手动触发：平均分配给所有 ≥S6 等级用户 |
| 销毁 | 1.5% | 发送至销毁地址（0x000...dEaD） |
| 官方钱包 | 0.5% | 直接发送 |
| 基金会 | 0.5% | 直接发送 |

> **S6+ 池分发规则**：累积到达某条件后，owner 调用接口，将池中所有 ETIM 平均分配给当时所有 ≥S6 等级的用户（谁先冲到 S6，累积的份额越多，越有优势）。

## 八、系统管理（Owner 权限）

| 功能 | 说明 |
|------|------|
| 设置入场金额范围 | min/max USD（6 位小数） |
| 设置每日释放率 | `dailyReleaseRate`，单位 1/1000 |
| 设置每日存款比例 | `dailyDepositRate`，单位 1/1000 |
| 设置等级团队阈值 | `levelConditions` 中的 `minTeamTokens` |
| 切换注入控制开关 | 储蓄池模式 on/off |
| 触发延迟注入 | 从储蓄池按量注入底池 |
| 更新每日价格 | 调用 `updateDailyPrice()` 更新 ETIM/ETH、USDC/ETH 价格快照 |
| 触发奖励分发 | 手动触发 5% ETH 奖励分配、S6+ 卖出税分配 |
| 开启/关闭交易 | `tradingEnabled`（买卖税 Hook） |
| 设置免税白名单 | `isExempt`（如 ETIMPoolHelper 地址） |

## 九、合约架构

```
ETIMToken.sol       — ERC20 代币，transfer 时回调 ETIMMain
ETIMNode.sol        — ERC721 节点 NFT（已部署，只读 balanceOf）
ETIMMain.sol        — 核心业务：入场、挖矿、等级、节点奖励
ETIMPoolHelper.sol  — Uniswap V4 流动性管理（swap / addLiquidity）
ETIMTaxHook.sol     — Uniswap V4 Hook：买卖税、卖出分配
```
